## 🎯 Цель проекта

Научиться писать **надёжные, поддерживаемые и документированные** API-тесты с использованием:
- `requests` — HTTP-клиент
- `pytest` — фреймворк для тестов
- `allure-pytest` — отчётность
- `jsonschema` — валидация ответов
- **Client Pattern** — архитектура тестов

**Итог**: Рабочий проект с 50+ тестами, отчётом Allure и интеграцией в CI/CD.

## 🚀 Быстрый старт

### 1. Клонируй репозиторий
```bash
git clone https://github.com/your-username/booker-api-tests.git
cd booker-api-tests
```

### 2. Установи зависимости
```bash
# Для РФ — с зеркалом
pip install -r requirements.txt \
    --index-url https://pypi.tuna.tsinghua.edu.cn/simple \
    --trusted-host pypi.tuna.tsinghua.edu.cn
```

### 3. Проверь установку
```bash
pytest --version
flake8 --version
allure --version
```

### 4. Запусти тесты
```bash
# Все тесты
pytest tests/ -v

# Только smoke-тесты
pytest tests/ -m smoke -v

# С отчётом Allure
pytest tests/ --alluredir=allure-results
allure serve allure-results
```

### 5. Отправь на проверку
```bash
git add .
git commit -m "feat: реализованы тесты Auth API"
git push
```
→ Открой вкладку **Actions** в GitHub — там будет отчёт автопроверки.

## 💡 Советы для успешной сдачи

1. **Начни с авторизации**: без токена не пройдут тесты на обновление/удаление.
2. **Используй фикстуры**: не создавай клиентов в каждом тесте — вынеси в `conftest.py`.
3. **Валидируй схемы**: это покажет понимание контрактов API.
4. **Логируй и документируй**: Allure-шаги помогут при ревью.
5. **Тестируй локально**: `pytest tests/test_auth.py -v` перед пушем сэкономит время.
6. **Чисти за собой**: удаляй тестовые брони в `finally` или фикстурах.
7. **Смотри Summary**: после каждого пуша проверяй отчёт — там подсказки по ошибкам.
